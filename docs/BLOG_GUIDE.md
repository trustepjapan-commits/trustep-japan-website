## 📦 ファイル構成

```
ホームページ/
├── blog.html                                     ← ブログ一覧ページ
├── blog/
│   ├── 2026-04-28-jinzai-kaihatsu-josei-kin-guide.html   ← 個別記事
│   ├── 2026-XX-XX-(slug).html                            ← 今後追加される記事
│   └── ...
├── index.html                                    ← トップページ06セクションに最新5本表示
└── docs/BLOG_GUIDE.md                            ← このマニュアル
```

### ファイル命名ルール（厳守）
```
blog/YYYY-MM-DD-スラッグ.html
例: blog/2026-04-28-jinzai-kaihatsu-josei-kin-guide.html
```
- 日付は **公開予定日** を入れる
- スラッグはアルファベット小文字＋ハイフン区切り（日本語禁止）
- このルールを守れば、自動ソート・URLが分かりやすい

## ✏️ 記事を新規追加する方法（依頼テンプレート）

Claude（私）に下記のテンプレで依頼すると、記事HTMLを生成→`blog/`に保存→`blog.html`一覧に追加→`index.html`トップ更新まで自動化します。

```

- カテゴリ：ai
- 公開日：2026-05-10
- 想定読者：中小企業の経営者・人事担当
- 入れたい論点：
  ① ○○
  ② ○○
  ③ ○○
- CTA：Claude Training Program
- 文字数：3000-5000字
- トーン：丁寧めの実務寄り
```

### 📝 依頼テンプレ③ 一括発注版（月初向け）
```
来月の記事15本書いて。

公開ペース：月水金の週3本
それぞれテーマ案を出して、OKなら全部一気に書いて
```

## 🎯 記事構造（テンプレ）

各記事は以下の構造で生成されます。すべて`<h2>`/`<h3>`/段落/箇条書き/コールアウト/数値ボックス/CTAカード/シェアバー/著者プロフィール/関連記事3件 を装備。

```
1. リード文（記事の要約と読者へのフック）
2. 本記事のポイント（コールアウト）
3. h2: 概要・前提知識
4. h2: 数値・事例で具体化（fact-box）
5. h2: 要件・条件
6. h2: 手順・フロー（番号付きリスト）
7. h2: 注意点・落とし穴（コールアウト）
8. h2: 専門家活用 or 自社施策の提案
9. CTAカード（無料相談へ）
10. h2: まとめ
11. シェアバー
12. 著者プロフィール
13. 関連記事3件
```

## 📥 旧WordPressからの記事移行（やる場合）

### ステップ1：WordPress管理画面から記事をエクスポート

1. WordPressにログイン
2. 左メニュー「ツール」→「**エクスポート**」
3. 「**投稿**」を選択
4. 「**エクスポートファイルをダウンロード**」をクリック
5. `trustep-japan.WordPress.YYYY-MM-DD.xml` のようなファイルがダウンロードされる

### ステップ2：そのXMLファイルを Claude（私）に渡す

```
このXMLファイルの記事を全部 blog/ に移行して
[XMLファイルを添付]
```

### ステップ3：私が一括変換

- WordPress記事を1記事1HTMLに変換
- 既存サイトのデザインに合わせて再構築
- 画像URLは旧サイトのまま参照（後日ダウンロードして自社サーバーに移動も可）
- カテゴリ・公開日・著者情報を保持
- 旧URLからのリダイレクト設定リスト（`.htaccess` or `_redirects`）も生成

### 移行時の注意点
- 100記事以上ある場合は分割で対応（XMLが大きいため）
- 画像数千枚規模になる場合は別途段取り（まずXMLだけ送ってください、適切な進め方をご相談します）

## ❓ よくある質問

**Q. 記事数が増えたら一覧ページが重くなりませんか？**
A. 100本を超えたら、自動でページネーション（10本ずつ表示）を実装します。「ページネーション入れて」と一言ご依頼ください。

**Q. アイキャッチ画像はどうしてますか？**
A. 現状はCSSグラデーションで自動生成（コスト・差し替え工数ゼロ）。実画像を入れたい場合は「○○の記事のアイキャッチを画像に差し替えて」とご依頼ください。

**Q. 記事の予約投稿はできますか？**
A. 静的HTMLなのでファイル名の日付＝公開日扱いです。`未来の日付ファイル`を作成しても即時公開されますが、`blog.html`一覧に「未公開」を出さない仕組みも用意可能です（ご希望時にお伝えください）。

**Q. 記事の中で動画やGoogle Mapsを埋め込めますか？**
A. はい、可能です。「YouTube動画 [URL] を埋め込んで」「Google Maps埋め込んで」と伝えてください。

**Q. 著者を「飛田 昌邦」名にしたいです**
A. 「全記事の著者を飛田名にして」または個別に「この記事の著者を飛田で」とご依頼ください。プロフィール画像も差し替え可能。
